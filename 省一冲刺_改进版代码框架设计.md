# 2026 大数据挑战赛省级一等奖冲刺：改进版代码框架设计文档

> 目标：不是照抄 baseline，而是设计一个**结构同样清晰、工程更规范、验证更可靠、算法更强**的比赛级代码框架。
>
> 本文档适合作为你后续重写项目代码的总设计说明，也可以改写成最终提交的 `README.md` 或项目开发规范。

---

## 0. 本项目要解决什么问题

本赛题的本质不是预测股票价格，而是：

```text
在每个预测日 T，根据 T 日及以前的沪深 300 历史数据，预测未来 T+1 开盘买入、T+5 开盘卖出的收益表现，最终选出最多 5 只股票并给出权重。
```

最终提交文件为：

```csv
stock_id,weight
600000,0.2
000001,0.2
...
```

所以我们的代码目标不是“做一个看起来复杂的模型”，而是建立一个完整、稳定、可复现的系统：

```text
数据校验 → 特征工程 → 标签构造 → 时间序列验证 → 多模型训练 → Top5 选股 → 权重分配 → 结果校验 → Docker 复现
```

---

## 1. 设计原则：比 baseline 更优秀的地方

baseline 的优点是结构清楚：

```text
config.py → utils.py → model.py → train.py → predict.py → result.csv
```

我们的框架也要保持这种清晰性，但不能只做“文件名更多”的表面升级。真正要升级的是下面这些点。

### 1.1 工程结构升级

baseline 把大量功能集中在 `utils.py` 和 `train.py` 中。我们的改进版要把功能拆清楚：

```text
数据校验归数据校验
特征工程归特征工程
标签构造归标签构造
验证回测归验证回测
模型训练归模型训练
推理输出归推理输出
```

这样后期调试、加特征、换模型、做融合都更容易。

### 1.2 验证体系升级

baseline 主要使用单次训练/验证划分。冲奖代码必须加入：

```text
滚动时间验证 walk-forward validation
多窗口回测
和 baseline 组合收益对比
Top5 收益统计
命中未来强势股票比例
最大亏损日统计
```

你不能只看一次验证结果。省一更看重稳定超过 baseline 的能力。

### 1.3 特征体系升级

baseline 已经有大量技术指标和 Alpha 特征，但仍然可以加强：

```text
横截面排名特征
市场状态特征
行业/风格相对强弱特征
异常波动过滤特征
停牌/缺失/低流动性处理
多周期收益和波动组合特征
```

比赛是从沪深 300 里选前 5，所以“横截面对比”非常关键。

### 1.4 模型体系升级

baseline 主要是单个 Transformer 排序模型。改进版不应只依赖一个深度模型，而应该采用：

```text
LightGBM Ranker
LightGBM Regressor
CatBoost / XGBoost
规则模型
轻量 Transformer / Temporal CNN
多模型 rank 融合
```

在股票数据这种高噪声场景里，单一复杂模型未必稳定。多模型融合往往比单模型更适合冲奖。

### 1.5 复现体系升级

从第一天开始就按提交要求写代码：

```text
固定随机种子
保存配置文件
保存特征列名
保存 scaler
保存模型版本
保存验证报告
保存最终推理日志
result.csv 自动格式检查
train.sh/test.sh 一键运行
Docker 离线可复现
```

## 3. 顶层运行脚本设计

比赛提交时，评审不会关心你内部代码多复杂，只关心：

```text
能不能训练
能不能预测
能不能复现
能不能生成合法 result.csv
```

所以顶层脚本必须简单稳定。

### 3.1 `init.sh`

作用：初始化必要目录和环境检查。

建议内容：

```bash
#!/bin/bash
set -e

mkdir -p ./output
mkdir -p ./temp/features_cache
mkdir -p ./temp/predictions_cache
mkdir -p ./reports

echo "[INIT] Project directories checked."
```

注意：正式复现环境不能联网，所以不要在 `init.sh` 里写下载数据、在线安装包等逻辑。

### 3.2 `train.sh`

作用：一键训练最终模型。

建议内容：

```bash
#!/bin/bash
set -e

python ./code/src/train.py \
  --config ./code/src/config/ensemble.yaml \
  --mode final_train
```

设计原则：

```text
不需要人工改路径
不需要打开 notebook
不需要手动运行多个脚本
所有模型训练流程由 train.py 统一调度
```

### 3.3 `test.sh`

作用：一键生成最终提交文件。

建议内容：

```bash
#!/bin/bash
set -e

python ./code/src/predict.py \
  --config ./code/src/config/ensemble.yaml \
  --model_dir ./model/final_ensemble \
  --output ./output/result.csv
```

运行后必须生成：

```text
./output/result.csv
```

## 6. 标签层设计

标签层是比赛成败的核心。标签错了，模型再复杂也没用。

### 6.1 `labels/make_label.py`

目标：构造官方评估一致的未来收益标签。

推荐实现逻辑：

```python
def make_future_return_label(df: pd.DataFrame, buy_shift=1, sell_shift=5) -> pd.DataFrame:
    """
    对每只股票构造未来收益标签：
    label = open_{T+5} / open_{T+1} - 1
    
    注意：shift 必须基于股票内部的交易日顺序，而不是自然日。
    """
    df = df.sort_values(["股票代码", "日期"]).copy()
    df["open_t1"] = df.groupby("股票代码")["开盘"].shift(-buy_shift)
    df["open_t5"] = df.groupby("股票代码")["开盘"].shift(-sell_shift)
    df["future_return_5d"] = df["open_t5"] / df["open_t1"] - 1
    return df
```

必须注意：

```text
不能用 T+1 到 T+5 的任何信息做特征
open_t1 和 open_t5 只能作为标签
最后 5 个交易日无法构造标签，要在训练阶段丢弃
```

### 6.2 `labels/make_relevance.py`

目标：把未来收益变成排序学习需要的 relevance。

推荐函数：

```python
def make_daily_relevance(df: pd.DataFrame, label_col="future_return_5d") -> pd.DataFrame:
    """
    每个日期内部，根据未来收益排序，生成排序标签。
    """
    pass
```

可以设计多种 relevance：

```text
rank_norm：当天收益排名归一化到 0~1
quantile_class：前 5% = 3，前 10% = 2，前 20% = 1，其余 = 0
topk_binary：未来收益前 5 或前 10% 记为 1，其余为 0
```

建议冲奖时同时保留：

```text

横截面排名 label
Top10% 二分类 label
```

这样你可以训练不同类型的模型。

### 7.4 `features/cross_section_features.py`

这是改进版框架必须加强的部分。

比赛是每天从沪深 300 中选相对最强的股票，所以横截面排名非常重要。

推荐函数：

```python
def add_cross_section_rank_features(df, feature_cols):
    """
    对每个日期，把指定特征在当天所有股票中做排名和分位数转换。
    """
    for col in feature_cols:
        df[f"cs_rank_{col}"] = df.groupby("日期")[col].rank(pct=True)
    return df
```

建议对这些特征做横截面 rank：

```text
过去 3/5/10/20 日收益
过去 5/10/20 日波动率
成交量放大倍数
换手率
均线偏离程度
RSI
MACD hist
BOLL 位置
```

横截面 rank 的优势：

```text
减少不同股票价格水平差异
让模型更关注相对强弱
和 Top5 选股目标更一致
对市场整体上涨/下跌更稳健
```

### 8.1 `datasets/tabular_dataset.py`

## 9.6 `models/rule_model.py`

规则模型不是用来最终单独提交，而是用来兜底和辅助融合。

推荐规则分数：

```text
score = 0.35 * 过去 5 日收益排名
      + 0.25 * 过去 20 日收益排名
      + 0.20 * 成交量放大排名
      + 0.10 * 均线趋势排名
      - 0.10 * 极端波动排名
```

作用：

```text
作为 baseline-like 稳定参考
当机器学习模型异常时提供兜底
可以加入融合，提升鲁棒性
```

## 10. 损失函数设计

如果使用深度排序模型，需要专门设计损失。

### 10.1 `losses/ranking_loss.py`

保留 pairwise + listwise，但加强 TopK。

推荐损失：

```text
loss = listwise_loss + α * pairwise_loss + β * topk_focus_loss
```

核心思想：

```text
全体股票排序要尽量正确
未来收益差异大的股票对更重要
未来 Top5 / Top10% 股票权重更高
```

### 10.2 `losses/topk_loss.py`

专门优化前几名。

建议：

```text
未来收益排名前 5 或前 10% 的股票，提高样本权重
预测 Top5 但真实很差的股票，加重惩罚
真实 Top5 但没被选中的股票，加重惩罚
```

不过不要为了复杂而复杂。深度模型只是辅助，主力建议仍然是 LightGBM Ranker。

## 12. 融合层设计

单模型容易不稳定，多模型融合是冲奖常用策略。

### 12.1 `ensemble/rank_fusion.py`

推荐用 rank 融合，而不是直接 score 融合。

原因：

```text
不同模型分数尺度不同
LightGBM 分数和 Transformer 分数不能直接相加
排名更稳定
```

推荐方法：

```python
def rank_average(prediction_list, weights):
    """
    对每个模型的预测分数转成当天横截面排名，再加权平均。
    """
    pass
```

示例：

```text
final_rank_score = 0.45 * lgb_ranker_rank
                 + 0.25 * lgb_regressor_rank
                 + 0.20 * transformer_rank
                 + 0.10 * rule_rank
```

### 12.2 `ensemble/model_selection.py`

不是所有模型都应该进入最终融合。

选择标准：

```text
验证收益稳定
多窗口胜过 baseline
和其他模型相关性不要太高
最差窗口不要太差
推理时间可控
```

如果某个模型单次分数高，但经常大亏，不建议进入最终融合。

## 13. 投资组合层设计

模型只输出股票分数，组合层负责把分数变成 `stock_id,weight`。

### 13.1 `portfolio/select_topk.py`

推荐函数：

```python

    """按最终分数选择前 top_k 只股票。"""
    pass
```

筛选前要检查：

```text
股票代码是否有效
是否有足够历史数据
是否存在极端异常价格
是否当日特征完整
是否预测分数为 NaN
```

### 13.2 `portfolio/risk_filter.py`

为了减少选到异常股票，可以做简单风险过滤。

建议过滤：

```text
最近 N 天停牌或数据缺失过多
最近一天振幅极端异常
最近波动率过高且模型置信度不足
成交量极低或成交额异常
```

不要过度过滤，否则可能错过强势股。

### 13.3 `portfolio/weight_allocator.py`

权重方案建议保留三种：

#### 方案 A：等权重

```text
5 只股票，每只 0.2
```

优点：稳定、简单、不容易出错。

#### 方案 B：分数加权

```text
模型分数越高，权重越高
但单只最大权重不超过 0.3
```

#### 方案 C：保守现金

```text
如果模型置信度不足，只买 3~4 只或保留部分现金
```

建议最终优先使用等权或轻微分数加权。过度集中容易放大风险。

def run_train_pipeline(config):

## 15. 预测主流程设计

### 15.1 `pipeline/predict_pipeline.py`

预测流程：

```text
1. 读取配置
2. 固定随机种子
3. 读取 data/train.csv
4. 数据检查和清洗
5. 使用训练时相同规则生成特征
6. 找到最新交易日
7. 构造候选股票集
8. 加载所有入选模型
9. 每个模型输出分数
10. 做 rank 融合
11. 风险过滤
12. 选择 Top5
13. 分配权重
14. 生成 output/result.csv
15. 校验 result.csv
16. 保存预测日志
```

推荐伪代码：

```python
def run_predict_pipeline(config, model_dir, output_path):
    set_seed(config.general.seed)

    raw_df = load_train_data(config.paths.train_file)
    validate_schema(raw_df)

    df = clean_ohlcv(raw_df)
    df = add_trading_index(df)

    feature_pipeline = FeaturePipeline.load(model_dir)
    df, feature_cols = feature_pipeline.transform(df)

    latest_date = df["日期"].max()
    candidate_df = build_latest_candidates(df, latest_date)

    model_predictions = []
    for model_info in load_ensemble_models(model_dir):
        pred = model_info.model.predict(candidate_df, feature_cols)
        model_predictions.append(pred)

    final_scores = rank_average(model_predictions, weights=config.ensemble.weights)
    final_scores = apply_risk_filter(final_scores, candidate_df)

    selected = select_topk_stocks(final_scores, top_k=config.portfolio.top_k)
    result = allocate_weights(selected, method=config.portfolio.weight_method)

    validate_result_csv(result)
    result.to_csv(output_path, index=False, encoding="utf-8")
```

## 21. 建议最终主线方案

对于零基础但目标省一的你，推荐最终主线不是先写深度模型，而是：

```text
强特征工程
+ LightGBM Ranker
+ LightGBM Regressor
+ 规则模型兜底
+ Rank 融合
+ Walk-forward 验证
+ 等权 Top5
+ 严格复现
```

原因：

```text
表格模型训练快
调参方便
更适合高噪声股票数据
复现风险低
更容易在 8 小时训练和 5 分钟推理限制内完成
```

深度模型可以作为加分项，但不建议作为唯一主模型。

## 22. 最终一句话

你的改进版代码不应该是“baseline 的重写版”，而应该是一个完整的比赛级系统：

```text
结构清楚得像 baseline，验证严谨超过 baseline，特征和模型比 baseline 更稳，工程复现比 baseline 更可靠。
```

真正冲省一的核心不是某一个神奇模型，而是：

```text
正确标签 + 强特征 + 稳定验证 + 多模型融合 + 合规复现
```
