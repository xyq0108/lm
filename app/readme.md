# 2026 大数据挑战赛 — 基于历史数据预测未来股价收益

## 项目简介

本项目基于沪深 300 指数成分股的历史股价数据，通过机器学习模型预测未来一周收益最大的股票组合（不超过 5 只，累计权重不超过 1）。

### 核心思路

```
强特征工程 + LightGBM Ranker + LightGBM Regressor + 规则模型兜底 + Rank融合 + Walk-forward验证 + 等权Top5 + 严格复现
```

### 比赛评估方式

- T+1 开盘买入 → T+5 开盘卖出
- 组合收益 = Σ (权重_i × 个股收益率_i) + 现金部分（收益率为 0）
- 最终排名按总收益率从高到低排序

## 快速开始

### 1. 环境准备

```bash
# 安装依赖
pip install -r requirements.txt

# 初始化目录
bash init.sh
```

### 2. 数据准备

将训练数据 `train.csv` 放在 `data/` 目录下。

数据需包含以下字段：
| 字段 | 说明 |
|------|------|
| 股票代码 | 6 位数字代码 |
| 日期 | 交易日 |
| 开盘 | 开盘价 |
| 收盘 | 收盘价 |
| 最高 | 最高价 |
| 最低 | 最低价 |
| 成交量 | 成交量 |
| 成交额 | 成交额 |
| 换手率 | 换手率 |
| 涨跌幅 | 涨跌幅 |

### 3. 训练模型

```bash
bash train.sh
```

### 4. 预测并生成结果

```bash
bash test.sh
```

输出文件: `output/result.csv`

### 5. Docker 复现

```bash
docker-compose up train   # 训练
docker-compose up predict # 预测
```

## 模型方案

### 主力模型
- **LightGBM Ranker (权重 0.50)** — LambdaRank 排序，目标 NDCG@5
- **LightGBM Regressor (权重 0.30)** — Huber 损失，预测收益幅度
- **规则模型 (权重 0.20)** — 多因子打分兜底

### 特征体系
| 类别 | 数量 | 说明 |
|------|------|------|
| 基础特征 | ~10 | K线形态、收益率、振幅 |
| 技术指标 | ~60+ | MA、MACD、RSI、KDJ、BOLL、ATR |
| Alpha 因子 | ~30+ | 趋势、动量、量价关系 |
| 横截面排名 | ~40+ | 每天内所有股票相对排名 |
| 市场状态 | ~8 | 整体市场环境 |

### 验证体系
- Walk-Forward 滚动时间验证（多窗口）
- 每日 Top5 组合收益统计
- 与 Baseline 对比胜率
